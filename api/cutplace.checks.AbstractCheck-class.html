Abstract check to be used as base class for other checks. The constructor should be called by descendants, the other methods do nothing and can be left untouched.

Create a check with the human readable description, a rule in a check dependent syntax which can act on the fields listed in availableFieldNames (in the same order as defined in the ICD) and the optional locationOfDefinition in the ICD. If no locationOfDefinition is provided, tools.createCallerInputLocation(["checks"]) is used.

reset(self)

source code 

Reset all internal resources needed by the check to keep track of the check conditions. By default do nothing.

It is recommended that the __init__() of any child classes calls this method.

This is called by interface.InterfaceControlDocument.validate() when starting to validate the data.

checkRow(self, rowMap, location)

source code 

Check row and in case it is invalid raise CheckError. By default do nothing.

RowMap is maps all field names to their respective value for this row, location is the tools.InputLocation where the row started in the input.

checkAtEnd(self, location)

source code 

Check at at end of document when all rows have been read and in case something is wrong raise CheckError. By default do nothing.

Location is the tools.InputLocation of the last row in the input.
